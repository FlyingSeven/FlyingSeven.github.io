关于hexo搭建

发表于 2016-08-25

这里主要记录下Hexo安装过程以及安装时遇到的问题，然后会有将Hexo部署到Github上的内容。所有操作在Mac OS上进行。部分网站链接可能需要翻墙才能看。

What is Hexo?

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

Requirements

首先，安装Hexo之前必须要先确认安装以下

  • Node.js
  • Git

测试Node.js是否安装

node -v

测试git是否安装 

git version

如果上述两个安装了的话，会显示出对应的安装版本，否则没有安装。

Installation

如果以上两个工具都已经安装了,直接使用npm安装Hexo

$ npm install -g hexo-cli

如果没有的话继续下面步骤

安装Git

Mac: Install it with Homebrew, MacPorts or installer.
这里我使用的是Homebrew, 按照API很容易实现。

安装Node.js

这里建议使用nvm 

$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh

验证是否安装成功：

command -v nvm

如果安装成功的话会输出’nvm’.然后重启terminal，运行cmd

$ nvm install stable

总体来说到这安装Hexo遇到的问题还是比较少的，一般会比较顺利。

SetUp

设置起来也很简单，运行下面几条cmd,其中folder为初始化的文件路径

$ hexo init <folder>
$ cd <folder>
$ npm install

初始化完成后，打开文件，可以看到大致如下的文件结构

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts //这个可能没有，后面使用draft介绍
|   └── _posts
└── themes
config.yml

bolg站点设置文件，可以参考configuration文件进行比较多的设置

package.json

应用数据，Markdown编辑器被默认安装。

scaffolds

Scaffold文件，你创建的每个post操作所要生成的文件都是基于scaffold里的文件格式。

source

要post的源文件，用于存放网站的文章内容。hexo会忽略掉隐藏的文件和以下划线（’__’）开头的文件（除了_posts文件）。可识别的文件（如Markdown，HTML）将被处理并添加进public文件，其他的则只会被简单的复制。

themes

Theme为主题存放的文件。后面会以nexT为例子做一些简单的配置。

查看效果

上面的都设置成功的话，现在可以先简单运行来看一下效果
1.生成静态文件 

hexo g

2.部署网站

hexo d

3.设置本地服务器

hexo s

然后会如下图所示

然后点击本地的路径http://localhost:4000/就可以查看当前的初始化blog。

写一篇Blog

新建一篇文章，使用以下cmd 

hexo new [layout] <title>

_如果layout没有设置的话，hexo会使用在_config.yml里的default_layout格式，如果title字段里包含空格的话，需要加上引号_。

然后根据layout的不同接下来会有不同的操作

post:

运行以下cmd

hexo g p

或者

hexo p g

两者效果是一样的，这时候开启下本地server

hexo s

点击 http://localhost:4000/就可以查看新加的文章，当然可以在source/_posts目录下找到新建的对应.md文件进行编辑后再查看。

draft

运行cmd

hexo publish [layout] <filename>

然后hexo会将drafts移动到source/_posts文件下,接下来就跟post的一样了，运行 

hexo g d

或者

hexo d g
